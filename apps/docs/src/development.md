### Tech Stack

- [SolidJS](https://www.solidjs.com/)
- [TailwindCSS](https://tailwindcss.com/)
- [DaisyUI](https://daisyui.com/)

### Preprocess

- Run `script/downloadFonts` to download web fonts used by Kiku.
- Adjust your Anki path at `script/copyDist.ts` and `vite.config.ts`.
- For packaging, prepare a deck named "Kiku" on your Anki.

#### Databases

::: info
`.db/_kiku_db_main.tar` and `.db/_kiku_db_main_manifest.json` have been excluded from `.gitignore` so you don't have to worry about them.
:::

`.db/_kiku_db_main.tar` and `.db/_kiku_db_main_manifest.json` are generated by `script/generateKikuDbMain.ts`,
which depends on other scripts such as `script/scrapJpdb.ts`, `script/scrapWk.ts`, `script/parseJmdict.ts`, `script/parseKanjiVg.ts`
to be run first.

### Build

::: info
Some of the scripts require Anki with AnkiConnect to be running.
:::

- `pnpm run build` to build the JavaScript and CSS bundles.
- `pnpm generateTemplate` to generate Front/Back/Styling template and post process the build output.
- `pnpm copyDist` to copy the generated files to your Anki `collection.media` directory.
- `pnpm updateNoteType` to update Kiku note type without re-importing `.apkg`.
- `pnpm apply` to run `build`, `generateTemplate`, `copyDist`, and `updateNoteType` in sequence.
- `pnpm package` to package the `.apkg`.

### Developing with Vite dev server

Simply run `pnpm run dev` to start the Vite dev server at port `5173`.
This will load `index.html` which has been made to mimic Anki's webview output.
The dev server also serves your Anki `collection.media` directory.

Switch between `http://localhost:5173/?side=back#` and `http://localhost:5173/?side=front#` to switch between the front and back side.
You can change the note fields by modifying `src/util/examples.ts`, you can visit Settings > Debug > AnkiFields to copy the fields.
